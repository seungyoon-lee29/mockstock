모든 티켓은 **T01 게이트 통과 + T03 배포 스켈레톤**을 공통 전제로 한다(선행 열에서는 생략).

**T02 분할로 단일 병목 해소**: `T02a→T04`, `T02c→T05`, `T02b→T08` — 각 다운스트림이 T02 서브티켓 **하나**에만 걸려 병렬 착수. 분할 전에는 셋 다 T02 전체(스키마+fillOrder+캘린더) 완료를 대기하는 단일 병목이었다.

**week1 과적재 완화**: 리플레이 시나리오 데이터 확보를 Week 1→Week 2(T06)로 이동 — 리플레이는 절삭 불가이나(§5.3) 데이터가 week1 병목이 되지 않도록.

**컷라인**(지연 시 절삭 순서, §5.2): 지정가(T08) → KIS 실시세(KR은 mock 유지, T05) → 봇 전략 다양화(T07). **리플레이(T09)·리더보드(T07)는 절삭 불가** — 이 둘이 없으면 제품 정체성이 사라진다.

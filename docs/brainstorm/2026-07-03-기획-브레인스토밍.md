# 모의 주식게임 — 기획 리셋 브레인스토밍 (시니어 기획자 에이전트)

> 현재 자산 파악: `web/`에 발견 홈(등락률 정렬 그리드), 큐레이션 유니버스 36종목(KR 18 + US 18, `web/src/lib/market/universe.ts`), 1초 랜덤워크 mock 시세 싱글톤(`priceSource.ts`) + SSE 스트림(`app/api/stream/route.ts`)까지 구현됨. 주문/포트폴리오/인증/worker는 미착수. 즉 "실시간 시세를 화면에 뿌리는 파이프"는 있고, "돈이 움직이는 경험"은 전무한 상태 — 기획 리셋의 적기.

---

## 1. 핵심 컨셉 후보 3개

### A. 「연습장」 — 진짜 같은 모의투자 워크스테이션
- **포지셔닝**: "실전 들어가기 전, 진짜 시세로 손에 익히는 토스 WTS급 연습장"
- **타깃**: 주식 시작 직전/직후 20~30대. 실계좌 잃는 게 무서운 사람.
- **왜 매력적인가**: 실시간 WS 시세 + 호가/체결 시뮬레이션이라는 기술적 코어가 그대로 제품 가치가 됨. 토스 벤치마크와 정합성이 가장 높고, "실제 증권사 UX를 얼마나 정교하게 재현했나"가 곧 완성도 지표라 포트폴리오 평가가 명확함. 단점: 차별화 스토리가 약해 '클론' 인상 위험.

### B. 「시즌 리그」 — 리셋되는 경쟁 게임
- **포지셔닝**: "매주 월요일 전원 1,000만 원 리셋, 금요일 마감 수익률로 랭킹 — 주식판 배틀로얄"
- **타깃**: 투자에 관심은 있지만 콘텐츠적 재미가 있어야 움직이는 게이머형 사용자, 커뮤니티/스터디 그룹.
- **왜 매력적인가**: 모의투자의 고질병(한 번 크게 잃으면 이탈, 고인물 복리 격차)을 시즌 리셋이 구조적으로 해결. 리더보드·시즌·티어라는 명확한 리텐션 루프가 있고, "게임 설계 능력"을 보여줄 수 있음. 실시간 시세가 '공정한 심판' 역할이라 기술 코어도 그대로 살림.

### C. 「타임머신」 — 과거 차트로 배우는 트레이딩 훈련소
- **포지셔닝**: "2020년 3월 코로나 폭락장에 떨어뜨려 놓고, 시간가속으로 10년치 매매 감각을 한 달에 압축"
- **타깃**: 이론은 아는데 '장세 경험'이 없는 학습 지향 사용자, 재테크 교육 콘텐츠 소비자.
- **왜 매력적인가**: 실시간 모의투자의 최대 약점 — "장 열릴 때만 재밌다, 결과 확인에 며칠 걸린다" — 를 정면으로 해결. 과거 데이터라 새벽에도 플레이 가능, 세션 하나가 15분짜리 완결 경험. 시나리오(닷컴버블, 리먼, 코로나)가 곧 콘텐츠라 확장성이 좋음. 단, 실시간 WS라는 기존 기술 자산과의 결이 다름(병행은 가능).

**기획자 추천**: B를 메인 스켈레톤으로, A를 기본 체력으로 깔고(리그도 결국 매매 UX가 좋아야 함), C의 '과거 리플레이'를 킬러 피처 하나로 편입하는 하이브리드.

## 2. 기능 아이디어 브레인스토밍 (27개)

### 트레이딩 코어
1. **간편매매 바텀시트** — 종목상세에서 금액/수량 토글 한 번에 시장가 체결 (토스 스타일, 3탭 이내 주문 완료)
2. **지정가 + 미체결 주문함** — 실시간 틱이 지정가 도달 시 자동 체결되는 매칭 엔진 (모의라서 가능한 '항상 체결' 정책 명시)
3. **호가창 시뮬레이션** — KIS 실시간 호가 그대로 노출, US는 스프레드 근사 생성. 슬리피지 개념 학습 장치
4. **평단·수익률 실시간 카드** — 보유 종목별 평가손익이 틱마다 숨쉬는 포트폴리오 홈
5. **매매일지 자동 생성** — 체결마다 "그때 왜 샀는지" 한 줄 메모 유도, 나중에 복기 타임라인으로
6. **환율 반영 통합 자산** — USD 포지션을 실시간 환율로 원화 환산, 환차손익 분리 표시 (KR+US 동시 지원 앱의 차별점)
7. **예약주문/분할매수 프리셋** — "3번에 나눠 사기" 버튼 하나로 물타기·분할매수 습관 학습

### 게임화·경쟁
8. **주간 시즌 리그** — 월요일 전원 동일 시드머니 리셋, 금요일 수익률 랭킹 확정, 시즌 뱃지 영구 보존
9. **티어 시스템** — 시즌 누적 성적으로 브론즈→다이아. 수익률만이 아니라 MDD·샤프비율 반영해 '존버 도박'과 '운영'을 구분
10. **데일리 미션** — "오늘 낙폭 상위 종목 1개 분석 메모 남기기" 같은 행동 미션으로 출석 대체
11. **최대낙폭(MDD) 페널티/생존 모드** — 계좌 -30% 도달 시 시즌 탈락하는 서든데스 룰 — 리스크 관리를 게임 규칙으로 내재화
12. **예측 스낵게임** — "삼성전자 오늘 오를까?" O/X를 장 시작 전 베팅, 장 마감에 정산 — 비장중 시간대 참여 장치
13. **시간가속 리플레이 모드** — 과거 실제 차트를 30배속으로 재생하며 매매, 세션 종료 시 실제 역사와 내 성적 비교 (컨셉 C의 코어)
14. **역사적 이벤트 시나리오 팩** — "리먼 사태 D-7에 입장했습니다" — 상황 브리핑 + 제한 시드로 도전, 클리어 조건 있음

### 소셜
15. **포트폴리오 공개 프로필** — 수익률 그래프 + 보유 섹터 도넛을 카드 이미지로 공유 (금액은 숨기고 %만)
16. **친구 리그(프라이빗 룸)** — 초대 코드로 5~10명 방 만들어 기간·시드 커스텀 대결 — 스터디/직장 동료 바이럴의 핵심
17. **따라하기(카피) 알림** — 상위 랭커가 매매하면 팔로워에게 체결 알림 (자동 카피매매는 하지 않음 — 모의라도 학습을 해침)
18. **매매 복기 피드** — 매매일지를 공개 설정하면 종목별 타임라인에 노출, "이 가격에 판 이유" 토론
19. **VS 배틀 공유 링크** — 시즌 종료 후 "나 vs 코스피 vs 친구" 3선 비교 그래프를 이미지로 뽑아주는 자랑 장치

### 학습·온보딩
20. **첫 매수 인터랙티브 튜토리얼** — 가입 직후 가상 1만 원으로 강제 첫 주문까지 90초 안에 완주시키는 온보딩 (aha moment 선행)
21. **용어 인라인 툴팁** — '평단', 'MDD', '호가' 등 화면 내 모든 전문용어에 탭하면 3줄 설명 (토스식 쉬운 언어)
22. **시즌 종료 리포트 카드** — "당신은 추격매수형입니다" — 매매 패턴 자동 분석 + 성향 라벨링 + 개선 팁 1개
23. **오답노트** — 손실 확정 매매만 모아 "샀을 때 vs 팔았을 때 차트"를 나란히 보여주는 복기 전용 뷰

### 리텐션
24. **장 마감 브리핑 푸시/카드** — "오늘 당신의 계좌 -1.2%, 코스피 -0.8%. 벌어진 이유 1줄" — 매일 돌아올 이유
25. **관심종목 가격 알림** — 지정 가격 도달 시 알림 → 재방문 트리거 (모의지만 실전 습관 형성)
26. **시즌 전적 아카이브** — 역대 시즌 성적·뱃지가 쌓이는 프로필 — 매몰비용 기반 리텐션
27. **주말 모드** — 장 안 열리는 주말엔 리플레이 모드·시나리오 팩·예측 게임을 전면 배치하는 홈 스위칭

## 3. MVP 컷 제안 (컨셉별 2주 데모 범위)

### A. 연습장 MVP
- **범위**: 발견 홈(현재 것 개선) → 종목상세(실시간 가격 + 경량 차트) → 시장가 간편매매 → 포트폴리오(보유/평가손익 실시간) → 관심종목. 인증은 익명 세션(localStorage 기반)으로 시작, DB는 테이블 3개(계좌·포지션·체결).
- **잘라내는 것**: 지정가, 호가창, 환율(1,400원 고정 상수), worker(현 mock 유지 → Finnhub 1개 마켓만 실연결).
- **데모 스토리**: "NVDA를 검색해서 100만 원 사고, 시세가 움직이며 수익률이 실시간으로 변한다."

### B. 시즌 리그 MVP
- **범위**: A의 매매 코어(시장가만) + 간단 로그인(구글 OAuth) + 주간 시즌 스케줄러(cron 1개: 월요일 리셋/금요일 확정) + 수익률 리더보드 1페이지 + 시즌 결과 공유 카드.
- **잘라내는 것**: 티어, 미션, 프라이빗 룸, MDD 지표(수익률 단일 기준).
- **데모 스토리**: "계정 2개로 접속해 서로 다른 종목을 사고, 리더보드에서 순위가 실시간으로 뒤집힌다." — 경쟁의 심장만 증명.

### C. 타임머신 MVP
- **범위**: 시나리오 1개(예: 2020년 코로나 폭락, 일봉 데이터 정적 JSON) + 재생 엔진(x1/x10/x30 배속, 일시정지) + 재생 중 매수/매도 + 세션 종료 리포트("실제 지수는 이후 +40% 반등, 당신은 -12%").
- **잘라내는 것**: 실시간 시세 전체(이 컨셉에선 불필요), 계정(세션 결과만 로컬 저장), 시나리오 2개 이상.
- **데모 스토리**: "15분 만에 폭락장 한 달을 겪고 성적표를 받는다." — 셋 중 데모 임팩트 최강, 기존 코드 재사용률 최저.

## 4. 오너에게 물어야 할 결정적 질문 5개

**Q1. 이 작품의 1차 관객은 채용 담당자인가, 실사용자인가?**
- (a) 채용 담당자 → 기능 폭보다 1~2개 화면의 디테일 완성도·기획 문서·지표 설계에 투자. 사용자 0명이어도 됨.
- (b) 실사용자 확보도 목표 → 온보딩·리텐션·바이럴 루프가 우선순위로 올라오고, 운영 비용과 시즌 운영 부담을 진짜로 져야 함.

**Q2. 계정/서버 상태를 어디까지 가져갈 것인가?**
- (a) 익명·로컬 우선 → 2주 데모 빠름, 그러나 리더보드·시즌·소셜 전부 봉인됨 (컨셉 B 사실상 불가).
- (b) OAuth + DB 처음부터 → 컨셉 B/소셜 가능, 대신 초반 1주가 인프라에 소모.

**Q3. 실시간 시세를 어느 수준까지 '진짜'로 할 것인가?**
- (a) mock 유지 + "실연동 아키텍처 문서"로 대체 → 개발 속도 최속, 데모는 24시간 살아있음.
- (b) Finnhub US만 실연결 → 중간값. 미국 장중(한국 밤)에만 데모가 살아있는 문제.
- (c) KIS까지 풀 연결 → 기술 어필 최대, 키 발급·토큰 갱신·worker 배포 운영 부담.

**Q4. 시즌 리셋 주기와 '파산'의 처리는?** (컨셉 B 계열 선택 시)
- (a) 주간 리셋 → 빠른 루프, 가벼운 몰입, 단기 도박성 매매 유도 위험.
- (b) 월간+상시 계좌 병행 → 진지한 운영 게임, 대신 신규 진입자가 중간 합류 시 불리.

**Q5. 규제·오해 리스크 라인을 어디에 긋나?**
- (a) 철저히 '게임' 프레이밍 → 랭킹·베팅 요소 자유롭지만 "투자 조장" 비판 여지, 예측 베팅(#12)은 사행성 인상 주의.
- (b) '교육' 프레이밍 → 리포트·오답노트·용어 학습이 전면에, 채용 시장(핀테크사)에서 더 안전한 스토리.

## 5. 포트폴리오 작품으로서 어필 포인트

1. **'왜'가 있는 기획 문서가 코드보다 먼저 보인다** — 컨셉 3개를 비교·기각한 이 문서 자체를 리포지토리에 남길 것. 트레이드오프를 명시적으로 결정한 흔적이 핵심.
2. **핀테크 도메인의 디테일 재현** — 호가 단위(KR 가격대별 호가가격단위), 상승 빨강/하락 파랑, 장 운영시간·프리장 처리, 환율 환산.
3. **실시간 데이터 파이프라인을 다뤄봤다는 증거** — Finnhub/KIS WS → worker → SSE 아키텍처 다이어그램 + "mock/실시세를 인터페이스 하나로 스왑"하는 `priceSource.ts` 설계 문서화.
4. **지표 설계 역량** — 첫 주문 도달률(활성화), 시즌 재참여율(리텐션), 공유 카드 클릭률(획득). 가입→첫 매수 퍼널.
5. **엣지케이스 사고력** — 장 마감 중 주문 처리, 시세 스트림 끊김 시 UX, 동시 체결 정합성을 '알려진 이슈와 결정' 문서로.
6. **한 개의 '와우' 순간에 집중** — 면접 데모 3분 안에 보여줄 시그니처 신 하나: (B라면) 실시간으로 순위가 뒤집히는 리더보드, (C라면) 30배속 폭락장 리플레이.
